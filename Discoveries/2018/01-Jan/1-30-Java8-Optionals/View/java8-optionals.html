Now null has some other issues besides uglifying our code by littering it with null checks. As nicely explained in a book I was reading recently on Java 8, null actually doesn't follow the statically typed system that Java uses1. It is the wrong way of representing the absence of a value.

Upgrades in Java 8 allow you to implement missing values in a statically typed manner with the Optional<T> class. The Optional object wraps a value, and it can be either present or absent (null). Now instead of performing null checks, you can simply interact with the Optional class.

In this discovery I am going to build an API for athletes in Java using the Optional method for null values. There are a couple of methods of the Optional class that we need to be familiar with before getting started:

static Optional<T> empty()

The empty() static method returns an empty Optional instance. This is the equivalent of a null value in a strictly typed setting.

static Optional<T> of(T value)

Wraps the argument value in the Optional object. However, if you pass null to the of() static method it will throw a NullPointerException. Only use of() if you know that the value being passed is not null.

static Optional<T> ofNullable(T value)

Performs the same operation as of() except if the value is absent no error will be thrown and an empty Optional object will be returned.
