Creating a Simple Geographical Map with Neo4j and Cypher


Assumed Knowledge:

Graph Databases Basic

Neo4j Basic

Cypher Query Language Basic


Lately I have been reading up on graph databases and their place in the NoSQL data storage universe. The graph database I've worked with is Neo4j, which is very easy to get set up. I've found the user interface to view graphs and type in queries to be very enjoyable and I highly recommend it if you need a graph database solution.

Without going into too much detail, the largest draw to graph databases is storing relationships between data and the speed at which you can query related data points (or in graph terms nodes/verticies). Relationships are first class citizens in graph databases which allows you to query related data by traversing the relationship itself. This is contrasted with a typical relational database solution where you have to find relationships through foreign keys or combine two tables of data with a very slow SQL JOIN operation. Our same slow query in a RDBMS (Relational DataBase Management System) is extremely quick in a graph database.

One of the first graphs I made in Neo4j was of the county I grew up in - Fairfield County CT. The first task on my list was to create a vertex to represent a state - in this case Connecticut. In Cypher (the query language used by Neo4j) that is easy!

We use the CREATE statement to build a vertex and pass it a label ':State' and a property 'name'. The label is used for grouping, in this case all states will have the label :State. You can also name the vertex along with supply additional key->value information in the verticies properties.
